# 安全策略

## 支持的版本

| 版本   | 是否支持 |
| ------ | -------- |
| 最新版 | ✅       |

## 报告漏洞

如果您发现 SecretPad 前端存在安全漏洞，请负责任地报告。

**请不要通过公开的 GitHub Issue 报告安全漏洞。**

### 报告方式

1. 通过 GitHub Security Advisories 报告： **Security → Advisories → Report a
   vulnerability**
2. 请包含：
   - 漏洞描述
   - 复现步骤
   - 潜在影响
   - 建议修复方案（如有）

### 响应时间

- **确认收到**：48 小时内
- **初步评估**：7 天内
- **修复时间**：严重 7 天 / 高危 14 天 / 中危 30 天

### 范围

以下在范围内：

- 渲染组件中的 XSS（跨站脚本）
- CSRF Token 绕过
- 客户端状态中的敏感数据泄露
- 不安全的 API 通信（缺少 TLS/认证头）
- 依赖漏洞（CVE ≥ 7.0）

### 不在范围

- 社会工程攻击
- 需要用户主动在控制台粘贴代码的 Self-XSS
- 非敏感页面的点击劫持
